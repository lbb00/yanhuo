/server 目录放置服务端代码，/app 目录使用 taro 开发的多端应用。这是一个厨房类的 app，需要实现手机号登录注册，食材、食谱资料库，食材食谱资料库应该设计的足够通用，像 wiki。app 上用户可以搜索、查看食谱、食材、帖子。发帖子记录自己做的某道菜，可以关联或创造食谱，当然也可以发一些无关紧要的东西，帖子要支持上传图片（dev 先存在本地）。发帖要注意隐私，可选是仅自己可见还是公开。帖子可以评论、回复。首页应该是推荐的 feed 流。个人主页应该有个类似 github 提交记录一样的东西，记录用户做饭的情况，用户可以看到自己会做了哪些菜，做过多少次。另外需要实现关注、粉丝列表之类的。我还希望再实现一个小工具，用户可以输入自己的现有食材，智能推荐菜谱。你需要为每个功能都书写 prd 或是知识库的东西，方便日后开发沉淀。
