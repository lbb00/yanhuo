# PRD-005: 智能选菜工具

## 1. 概述
帮助用户解决“不知道吃什么”的难题。用户输入现有食材，系统推荐包含这些食材的食谱。
初期采用简单的标签匹配算法，未来可升级为 AI 模型推荐。

## 2. 用户故事
- **输入食材**：
  - 我可以输入多个食材名称（如“鸡蛋”、“番茄”）。
  - 系统支持模糊搜索或自动补全食材。
- **获取推荐**：
  - 点击“开始匹配”，系统返回一个食谱列表。
  - 列表按匹配度排序（包含越多输入食材的食谱越靠前）。
- **查看详情**：
  - 点击推荐的食谱进入详情页查看做法。

## 3. 核心功能

### 3.1 匹配算法 (V1)
- **输入**：食材 ID 列表 或 名称列表。
- **逻辑**：
  1. 查找所有包含至少一个输入食材的食谱。
  2. 计算每个食谱的匹配分数：
     - 分数 = (食谱中包含的输入食材数量) / (食谱所需总食材数量) * 权重 + (包含的输入食材数量) * 权重
  3. 排序返回。

### 3.2 界面交互
- 简单的输入框，支持回车添加 tag。
- 结果列表展示。

## 4. API 接口设计

### 4.1 推荐接口
- `POST /smart/recommend`
- **Body**: `{ ingredients: ["鸡蛋", "番茄"] }`
- **Response**:
  ```json
  [
    {
      "id": 1,
      "title": "番茄炒蛋",
      "matchCount": 2,
      "missingCount": 1, // 还需要葱花
      "score": 0.9
    }
  ]
  ```
